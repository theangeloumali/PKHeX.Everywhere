# Product

## Register

product

## Users

Pokemon players and existing PKHeX users who need to inspect and edit save files from a browser on desktop or mobile. They value direct access to save-compatible Pokemon, inventory, boxes, and export tools without repeatedly navigating through low-level editor steps.

## Product Purpose

PKHeX Everywhere brings trusted PKHeX capabilities to cross-platform web and terminal interfaces. The web product succeeds when users can safely find, understand, and apply common save edits quickly while preserving PKHeX.Core compatibility and legality constraints.

## Brand Personality

Familiar, efficient, trustworthy. The interface should feel like a focused utility: clear enough for occasional users, dense enough for experienced PKHeX users, and restrained enough that the workflow remains more prominent than decoration.

## Anti-references

- A neon gaming dashboard that prioritizes visual effects over data clarity.
- A desktop-form clone that reproduces cramped WinForms controls without adapting them for browsers or touch.
- A generic card-heavy dashboard that hides frequent actions behind decorative containers.
- Destructive or bulk actions with unclear scope, silent partial completion, or implicit overwrite behavior.

## Design Principles

1. Put the user's current save context and available actions in view before asking for input.
2. Make frequent edits direct; reserve detailed editors for choices that genuinely need them.
3. Preserve safe, compatible defaults and explain every skipped or partial operation.
4. Reuse familiar PKHeX language while adapting interactions for keyboard, touch, and responsive layouts.
5. Keep existing workflows working while introducing faster paths alongside them.

## Accessibility & Inclusion

Target WCAG 2.2 AA. All workflows must be keyboard-complete, use visible focus indicators, label form controls, announce validation and operation results, avoid color-only state, support reduced motion, and remain usable at mobile widths and 200% zoom.
